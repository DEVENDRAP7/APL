The Challenge
What breaks at 72,000 people

Every large event faces the same systemic failures — and they all compound each other at scale.

🚶‍♂️

Crowd Bottlenecks

Entry gates, concourses, and exits become dangerously congested with no real-time intelligence to redistribute flow.

⏳

Long Wait Times

Concession queues of 15–25 minutes with no dynamic staffing response or attendee guidance to alternatives.

📻

Siloed Staff Comms

Security, medics, ushers, and logistics operate on separate channels with no unified coordination layer.

🚨

Slow Emergency Response

Medical incidents and safety threats take minutes to escalate — time that costs lives in critical situations.

🗺️

No Personal Guidance

Attendees navigate 72,000-person venues alone — no smart assistant for seat finding, food orders, or accessibility.

🚗

Post-Game Chaos

Exit surges overwhelm parking lots and transport with no predictive management or dynamic routing.

The 3 AI Agents

Only the decisions that genuinely require AI reasoning. Everything else is fast, rule-based code.

💬
Attendee Experience Agent
claude-sonnet-4-6 · Streaming

Handles every fan's personal query in natural language — seat finding, accessible routes, food ordering, lost & found, live schedule updates. Streams responses in real time to the mobile app.

lookup_seat get_queue_wait find_nearest_amenity place_food_order request_accessibility_help
🚨
Emergency Response Agent
claude-opus-4-7 · Adaptive Thinking

Safety-critical reasoning for medical incidents, security threats, fires, and evacuations. Uses Opus 4.7 because every emergency is unique — no rule set covers every combination of simultaneous incidents.

get_venue_state get_evacuation_routes activate_pa_announcement contact_emergency_services trigger_zone_evacuation dispatch_medical_team
👁️
Vision Anomaly Agent
claude-sonnet-4-6 · Vision

Interprets flagged camera frames using Claude's vision capability. YOLO runs every 5 seconds on all 96 cameras for free. When anomaly_score exceeds 0.7, Claude Vision determines what's actually happening.

analyse_frame assess_safety_risk recommend_action

Architecture
How it works

Five layers from physical hardware to real-world action — only 3 of them use AI.

1

Physical Sensors

284 IoT sensors + 96 CCTV cameras across the venue publish data every 2–5 seconds via MQTT and RTSP streams.

2

Ingestion Pipeline

Raw sensor data is normalised, deduplicated, threshold-gated and enriched with zone metadata before entering the system.

3

Rule Engine

80% of decisions — crowd routing, queue alerts, staff dispatch, parking overflow — are instant rule-based code. No API cost.

4

AI Agents

The 3 Claude agents handle only what rules can't: natural language chat, novel emergency reasoning, and visual anomaly interpretation.

5

Real-World Action

PA broadcasts, digital signage, staff dispatch, mobile push notifications, and emergency service calls — all automated.
